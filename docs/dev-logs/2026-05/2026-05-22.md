## 考えたこと

昨日の検討を踏まえ、個人情報や法的リスクが重くなりすぎる題材ではなく、安心して公開できる BtoC アプリとして Tsumoo を再出発させることにしました。

Tsumoo では、単純な ToDo CRUD ではなく、入力、完了、振り返りの体験を丁寧に設計します。

要件定義では、画面名を「今日のToDo一覧」に固定せず、ToDo 一覧の中に日付条件フィルターを持たせる方針にしました。昨日登録した ToDo を今日完了するケースも自然に扱えるようにするためです。

## 決めたこと

- サービス名は Tsumoo（つもー）とする
- コンセプトは「今日の『できた』を積もう。」とする
- アプリ本体は非公開リポジトリで管理する
- 公開ログは GitHub Pages で公開する
- フェーズ1ではチェック型 ToDo と軽量なステータス型 ToDo を扱う
- 認証は Google 認証とメールアドレス / パスワード認証を扱う
- メールアドレス / パスワード認証では確認メールとパスワードリセットを扱う
- ToDo 一覧では、すべて / 未完了 / 完了 のフィルターを基本にする
- 日付条件は、実施予定（開始） / 実施予定（終了） / 期限 を切り替えられるようにする
- 今日の達成状況は、作成日ではなく完了日を基準に集計する
- ステータス型 ToDo の候補は、未着手 / 進行中 / 完了 / 保留 を基本にする
- アカウント設定では、メールアドレス変更、パスワード変更、ログアウト、退会を扱う
- フェーズ1のタスクは、設計・開発・テスト・公開ログ整備を含めて30件に整理する
