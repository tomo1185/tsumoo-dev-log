# 2026年7月6日（月）

## 今日やったこと

- **TSUMOO-55 カテゴリ機能に向けたタスク管理設計整理**を進めた
  - カテゴリ機能はフェーズ1画面には出さず、将来拡張に向けた設計整理として扱う方針にした
  - 「カテゴリ」よりも、自分だけで作って管理できる **個人用タスクグループ** に近い機能として考えることにした
  - 1タスクに対してグループは1つ、または未設定という形を基本方針にした
  - グループには名前、色、表示順を持たせる方向にした
  - グループ管理はタスク管理画面に埋め込まず、サイドメニューから独立した管理画面として扱う方針にした
- タスク管理画面の Next.js 側を大きく整理した
  - `useTaskList` に集まっていた取得、状態管理、ページング、mutation、完了フィードバック、loading状態を小さく分けた
  - `task-list` 配下を `fetch`、`mutations`、`state`、`ui`、`controls` の責務に分けた
  - grouped fetch の責務を分け、task group contract に依存する箇所を追いやすくした
  - `TaskListFilters` を詳細検索、日付フィルタ、ステータスフィルタなどに分けた
  - `TaskListSections` を主要セクション、完了関連セクション、日付なしセクション、表示部品に分けた
  - `TaskListItem` と `TaskListMessages` を分離し、一覧表示の部品構成を読みやすくした
- Laravel 側の整理にも着手した
  - `ListTasksAction` に集まっていた query 作成、日付分類、group_counts、review summary 集計の責務を分け始めた
  - group_counts と review summary の集計境界を確認し、タスク一覧と振り返りで同じ分類を参照しやすくした
  - 対象日、予定外、関連完了などの日付分類を、後から追いやすい Service へ分ける方針にした
- ドキュメントを更新した
  - TSUMOO-55 のタスク開始メモに、整理済みの内容と残りの確認観点を追記した
  - フロント側の分離内容を、あとから追いやすい単位で記録した
