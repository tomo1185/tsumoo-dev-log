  - 機能実装ガイドの作成マニュアルとテンプレートを追加
  - アカウント登録を題材に、`auth-register` の実装ガイドを作成
  - hook、画面、API、Laravel 側処理の対応関係を追えるように整理
- 保守性向上のためのコメント方針を調整

## 考えたこと

今日は、Tsumoo で初めて Laravel 側の実処理に踏み込む日になりました。

登録処理は、画面だけで完結する機能ではなく、DB、メール、認証、バリデーション、画面遷移が一気につながるため、設計書とコードを行き来しながら慎重に進めました。

また、実装を進める中で「あとから人間が読めるか」がかなり重要だと感じました。画面、hook、API 通信、Laravel controller、request、model を分けたことで、処理の流れを追いやすくなりました。

機能実装ガイドを作ったことで、設計書とコードの間にある「読むための地図」ができたのも大きな収穫でした。

## 決めたこと

- 登録フォームの状態と送信処理は hook にまとめる
- API 通信処理はフォーム hook から分ける
- API エラーはトーストではなく、フォーム上部または対象項目近くに表示する
- 登録成功後は確認メール送信完了画面へ遷移する
- 確認メール送信完了画面は、登録導線から来た場合だけ表示する
- メール確認は署名付き URL で処理する
- 初期 migration は、ファイル名と中身が一致するように目的ごとに分ける
- 今後の機能実装では、機能実装ガイドをあわせて育てる
