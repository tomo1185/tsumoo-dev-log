## 考えたこと

今日は、アカウント登録処理に入る前に、設計書と実装方針のすり合わせを行いました。

認証まわりは後から直すのが大変なので、実装だけを先に進めるのではなく、CSRF、Cookie、メール確認リンク、登録後の画面遷移を一つずつ確認しました。

特に、確認メール送信完了画面はセキュリティ上の問題だけでなく、ユーザー心理として「直接見えてよい画面なのか」という観点も大事だと感じました。

## 決めたこと

- Tsumoo の通常ログインは Cookie + CSRF を基本方針にする
- POST / PATCH / DELETE では CSRF cookie を取得してから API を呼ぶ方針にする
- 登録成功後は確認メール送信完了画面へ遷移する
- 確認メール送信完了画面は、登録導線から来た場合だけ表示できるようにする
- メール確認リンクは、署名付き URL と id / hash の検証で処理する方針にする
- Google 認証と Google カレンダー連携は、通常ログインとは分けて設計する
