## 補足

タスク数は、設計や公開運用の整理に合わせて増減する可能性があります。

画面コーディングを先に一通り進める方針に変更したため、TSUMOO-30 以降は画面実装タスクを先に並べています。認証処理や登録処理などの実処理は、画面コーディング後の後続タスクとして整理します。

開始予定日と期限は、実装中に見えた認証まわりの追加対応を反映し、少し余裕を持たせて見直しています。

デプロイ方式は、運用負荷と安全性のバランスを見ながら整理します。
